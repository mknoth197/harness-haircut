# Project standards — harness-haircut

> This is the **canonical** source of truth for contributing to this repo. Every per-provider file (`CLAUDE.md`, `GEMINI.md`, `.github/copilot-instructions.md`) is a hand-emitted projection of this document plus the files under [`.agents/instructions/`](.agents/instructions/). Once `harness-haircut apply` ships ([C2 #12](https://github.com/mknoth197/harness-haircut/issues/12)), those projections are regenerated by the tool and the hand-emit step goes away. Edits should land here first.

## What this project is

`harness-haircut` is a CLI tool that audits a single repository and consolidates redundant AI-provider configuration files (`AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, `.github/copilot-instructions.md`, …) into a single canonical source of truth, then projects that source into each provider's native format. Full product context lives in [`docs/PRD.md`](docs/PRD.md). Planned work lives in [`docs/stories/`](docs/stories/) (one EARS user story per planned GitHub issue).

## Tech stack

- **Node.js 24+** (see [`.nvmrc`](.nvmrc), `engines.node` in [`package.json`](package.json))
- **TypeScript** in strict mode, ESM, NodeNext module resolution (see [`tsconfig.json`](tsconfig.json) for typecheck/editor scope; [`tsconfig.build.json`](tsconfig.build.json) for emit)
- **No runtime dependencies** in v1 (per [PRD goal 5](docs/PRD.md): zero AI-provider runtime deps; we extend that to zero npm runtime deps where reasonable)
- **Tests** via `node --test` with native type-stripping; no Jest/Vitest/Mocha
- **Lint** via ESLint flat config with `typescript-eslint`
- **Distribution** via `npx` (npm); `uvx` (Python wheel) is post-v1 per [PRD §15](docs/PRD.md)

## Architecture

Clean / Hexagonal, four layers, dependencies point **inward**. Full detail in [`.agents/instructions/software-architecture.md`](.agents/instructions/software-architecture.md).

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 4 — Composition root  (src/bin.ts, src/cli.ts)       │
│  ────────────────────────────────────────────────────────   │
│  ▼                                                          │
│  Layer 3 — Adapters / Gateways  (src/adapters/, src/gateways/) │
│  ────────────────────────────────────────────────────────   │
│  ▼                                                          │
│  Layer 2 — Use cases  (src/use-cases/)                      │
│  ────────────────────────────────────────────────────────   │
│  ▼                                                          │
│  Layer 1 — Entities / IR  (src/entities/)                   │
└─────────────────────────────────────────────────────────────┘
```

Inner layers MUST NOT import from outer layers. This is enforced by code review today and by a dependency-cruiser rule once F1 lands.

## Testing

Three categories — full detail in [`.agents/instructions/testing.md`](.agents/instructions/testing.md):

1. **Unit** — per layer, single file under test, no I/O.
2. **Integration** — use case + real filesystem in `os.tmpdir()`; no provider runtimes.
3. **End-to-end** — `spawn` the built `dist/bin.js` and assert on exit codes + stdio.

All test files: `test/**/*.test.ts`. Run with `npm test`.

## Commit & PR style

Conventional Commits, full detail in [`.agents/instructions/commit-style.md`](.agents/instructions/commit-style.md). Short version:

- Subject ≤72 chars: `<type>: <imperative subject>` where type ∈ {feat, fix, docs, refactor, test, build, ci, chore}.
- Body explains *why*, not *what*.
- Footers for issue refs (`Refs: #N`, `Closes: #N`) and `Co-Authored-By:`.
- One commit per logical change. Squash on merge is fine if commits are noisy.

## Definition of Done

A change is done when:

- [ ] Implementation matches the corresponding EARS user story in [`docs/stories/`](docs/stories/).
- [ ] `npm run lint`, `npm run typecheck`, `npm test` all pass locally.
- [ ] CI green on the PR (see [`.github/workflows/ci.yml`](.github/workflows/ci.yml)).
- [ ] PR description explains alignment with the story's EARS rules and the PRD.
- [ ] Public types and `run()` behavior remain backward-compatible *within v0.x*.

## Quick command reference

```sh
npm ci            # clean install
npm run build     # tsc -p tsconfig.build.json -> dist/; chmod +x dist/bin.js
npm run lint      # ESLint with type-aware rules
npm run typecheck # tsc --noEmit across src/, test/, scripts/
npm test          # build, then node --test against test/**/*.test.ts
npm pack          # produce harness-haircut-<v>.tgz for end-to-end smoke
```
