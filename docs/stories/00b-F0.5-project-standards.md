# F0.5 — Project standards & dogfood AGENTS.md

**Type:** Foundational (extends F0)
**Depends on:** F0
**Blocks:** every subsequent code-producing story (F1+)
**Labels:** `enhancement`, `foundation`, `dogfood`

## Context
F0 stood up the scaffold but said nothing about *how* the code should be structured, tested, or contributed. Before the first business-logic story (F1) lands, codify the standards so contributors and AI agents enforce them uniformly. This story also dogfoods our own canonical format: `AGENTS.md` at root + `.agents/instructions/` per-topic — exactly what [PRD §8](../PRD.md) specifies. Until `apply` ships (C2 [#12](https://github.com/mknoth197/harness-haircut/issues/12)), the per-provider files (`CLAUDE.md`, `GEMINI.md`, `.github/copilot-instructions.md`, `.github/instructions/*.instructions.md`) are hand-emitted projections; once apply lands, those are deleted and regenerated by the tool.

## Requirements (EARS)

- **U1.** The repo shall contain a root `AGENTS.md` as the single source of truth for project standards.
- **U2.** Per-topic canonical instruction files shall live under `.agents/instructions/<topic>.md` with `scope:` frontmatter declaring where the rules apply.
- **U3.** Per-provider projections shall exist for Claude, Gemini, and Copilot, each marked as hand-emitted-pending-apply.
- **U4.** The repo shall publish standards for: language & toolchain, CLEAN architecture layers, testing layout, commit style, and dependency direction.
- **U5.** Editor + Node version conventions shall be machine-readable: `.editorconfig`, `.nvmrc`.
- **EV1.** When a contributor opens any `src/**/*.ts` file in an editor with Copilot, the architecture instructions shall be in scope via Copilot's `applyTo` glob.
- **EV2.** When `harness-haircut apply` lands (C2), the per-provider projections shall be regenerable from the canonical sources, demonstrating round-trip parity.
- **UN1.** If a per-provider projection drifts from its canonical source, then a future `audit` run shall report it as drift (validated by C1 [#11](https://github.com/mknoth197/harness-haircut/issues/11)).

## Acceptance criteria

- [ ] `AGENTS.md` at repo root.

- [ ] `CLAUDE.md`, `GEMINI.md`, `.github/copilot-instructions.md`, and `.github/instructions/*.instructions.md` exist as hand-emitted projections with a header noting they will be regenerated by `harness-haircut apply` once it ships.
