# HH-W001 — lossy glob downgrade

**Severity:** warn

## What fires it

A canonical `scope:` glob uses syntax the target provider's matcher language
cannot express (e.g. brace expansion or negation projected into Copilot's
`applyTo`). The adapter emits the closest expressible glob and names the
downgraded shape in the warning.

## What to do

Check the downgraded glob reported in the warning. If it over- or
under-matches for that provider, simplify the canonical glob to the common
subset every enabled provider understands.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W001"] }
```
