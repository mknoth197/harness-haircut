# HH-W003 — hook event unmappable for a provider

**Severity:** warn

## What fires it

A canonical hook uses an event the target provider's taxonomy lacks (the
provider's entry in the event mapping table is `null`). Examples: Codex has
no `session-end` event; Gemini CLI has no `subagent-start`/`subagent-stop`
events. The hook is skipped **for that provider only** and projected
normally everywhere else.

## What to do

Nothing, if the gap is acceptable — the warning exists so the gap is never
silent. Otherwise, restructure the hook around an event all enabled
providers support (see `src/adapters/event-maps.ts` for the tables).

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W003"] }
```
