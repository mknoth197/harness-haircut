# HH-W004 — provider size cap exceeded

**Severity:** warn

## What fires it

Projected instruction content exceeds a documented provider limit — most
notably Codex's 32 KiB cap on the combined nested `AGENTS.md` chain
(`project_doc_max_bytes` default). Content past the cap is silently
truncated by the provider, not by harness-haircut.

## What to do

Keep the root `AGENTS.md` lean. Push detail into per-topic files under
`.agents/instructions/` (linked, not inlined) so the combined chain stays
under the cap.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W004"] }
```
