# HH-W005 — duplicate hook sources detected in provider config

**Severity:** warn

## What fires it

A provider already defines hooks in a location harness-haircut does not own,
in addition to the file it projects — e.g. a `[hooks]` table in
`.codex/config.toml` while the tool emits `.codex/hooks.json`. Both sources
would be active at once, so hooks could fire twice.

## What to do

Move the hand-written hook definitions into canonical
`.agents/hooks/<event>.<name>.<ext>` files and delete the duplicate provider
config, or remove the canonical hooks if the provider-local ones are
intentional.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W005"] }
```
