# HH-W005 — conflicting or duplicate provider configuration source detected

**Severity:** warn

## What fires it

A provider config source exists that conflicts with — or duplicates — what
harness-haircut projects. Two shapes:

- **Duplicate hook sources**: a provider already defines hooks in a location
  harness-haircut does not own, in addition to the file it projects — e.g. a
  `[hooks]` table in `.codex/config.toml` while the tool emits
  `.codex/hooks.json`. Both sources would be active at once, so hooks could
  fire twice.
- **Conflicting instruction sources**: a provider context file the tool would
  emit as a one-line `@AGENTS.md` import shim (`CLAUDE.md`, `GEMINI.md` in
  shim mode) already exists with hand-written content. The file is left
  untouched, but it competes with `AGENTS.md` as an instruction source.

## What to do

For duplicate hooks: move the hand-written hook definitions into canonical
`.agents/hooks/<event>.<name>.<ext>` files and delete the duplicate provider
config, or remove the canonical hooks if the provider-local ones are
intentional. For conflicting instruction files: fold the hand-written content
into `AGENTS.md` (or below the import line) — `init` (C3) resolves this
interactively.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W005"] }
```
