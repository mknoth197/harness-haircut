# HH-W006 — deprecated provider config key detected

**Severity:** warn

## What fires it

An existing provider config uses a key its provider has deprecated — e.g.
Gemini CLI's flat `contextFileName` (v1 settings schema), superseded by the
nested `context.fileName` key. harness-haircut writes only the current
schema and leaves the deprecated key in place.

## What to do

Remove the deprecated key from the provider config once you have confirmed
nothing else depends on it; the projected current-schema key replaces it.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W006"] }
```
