# HH-W007 — canonical surface unrepresentable for provider

**Severity:** warn

## What fires it

A canonical artifact has **no projection target at all** for a provider — not
even a lossy approximation. Example: a scoped instruction fragment
(`.agents/instructions/*.md`) projected to Gemini CLI, whose context files
have no path-scoping mechanism (no `paths:`/`applyTo` equivalent). The
artifact is skipped for that provider only; every other enabled provider
still receives it.

This differs from `HH-W001` (lossy glob downgrade): W001 means the content
was emitted with a degraded scope, W007 means the provider cannot carry the
content at all.

## What to do

Decide whether the affected provider needs the content. If it does, move the
fragment's rules into `AGENTS.md` (which every v1 provider consumes) — at
the cost of losing the path scoping for all providers. If the gap is
acceptable, suppress the code.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W007"] }
```
