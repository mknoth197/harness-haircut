# HH-W010 — unknown attachment under `.agents/`

**Severity:** warn

## What fires it

The parser found a file under `.agents/` that is not part of the canonical
layout (`instructions/<name>.md`, `skills/<name>/…`,
`hooks/<event>.<name>.<ext>`). The file is carried through the IR as an
opaque attachment — it is never deleted or modified — but no provider
projection is derived from it.

## What to do

If the file was meant to be canonical, move it to the right place (e.g. a
skill folder needs a `SKILL.md` entrypoint; instruction fragments must sit
directly in `.agents/instructions/` with a `.md` extension). If it is
intentional scratch content, gitignore it or accept the warning.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W010"] }
```
