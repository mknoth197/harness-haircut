# HH-W011 — frontmatter in AGENTS.md

**Severity:** warn

## What fires it

A root or nested `AGENTS.md` begins with a YAML frontmatter block. The
agents.md spec defines no frontmatter, and every native consumer (Codex,
Copilot, configured Gemini CLI, …) injects the file **verbatim** — the block
leaks as raw text into provider prompts. The parser treats the block as
literal markdown content; it is not interpreted.

## What to do

Remove the frontmatter from `AGENTS.md`. If you need scope metadata, use a
fragment under `.agents/instructions/<name>.md` with a `scope:` frontmatter
key — that directory is read by no provider natively, so frontmatter is safe
there.

## Suppression

Add the code to `harness-haircut.config.json`:

```json
{ "suppressWarnings": ["HH-W011"] }
```
