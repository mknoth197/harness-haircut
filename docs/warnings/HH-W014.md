# HH-W014 — unquoted frontmatter value contains an ambiguous " #"

**Severity:** warn

## What fires it

A frontmatter value that is **not cleanly quoted** contains a ` #` (a space
followed by a hash). A strict YAML parser reads everything from the ` #`
onward as a trailing comment and drops it, but the real consumer (Claude
Code) reads the whole value verbatim. The two interpretations disagree, so
the value is ambiguous.

The parser matches the consumer: it keeps the **full text** literal (the
round-trip stays byte-stable) and mints this warning instead of guessing.
Every value shape is covered — a bare scalar, a block-sequence item, and an
inline-array item all warn:

```yaml
scope: src/** # only the source tree     # warned — kept as "src/** # only the source tree"
paths: [src/** # note, test/**]          # warned — first item kept literal
notes:
  - see issue #12 for context            # warned — kept literal
```

A **cleanly quoted** value never fires this — inside matching quotes, YAML
cannot open a comment, so ` #` is unambiguously literal and parses with no
warning (`description: "fixes issue #12"`).

Why this matters: `scope:` (and its `paths:` / `applyTo` projections to
Claude and Copilot) is a glob. A scope like `src/** # note` kept literal
projects to `paths: ["src/** # note"]` — a glob that silently never matches.
The warning makes the broken glob visible rather than letting it ship
unnoticed. This is a warning, not an error: a working skill or instruction
must not abort the whole audit over a comment-like value (#58, #60).

## What to do

Quote the whole value so the intent is explicit for every reader, or reword
it to drop the ` #`:

```yaml
scope: "src/** # only the source tree"   # literal, unambiguous
scope: src/**                            # or drop the trailing note entirely
```

Quotes do not nest in the supported subset (no escape processing), so the
advisory steers you to a style the value can actually wear:

- value already contains a `"` → single-quote it (or reword),
- value already contains a `'` → double-quote it (or reword),
- value contains both quote characters → reword it to drop the ` #` (neither
  quote style can wrap it faithfully).

## Suppression

There is no warning-suppression mechanism in v0.x (the `suppressWarnings`
config key mentioned by some sibling pages is not implemented). HH-W014
clears when the value is quoted or reworded so it is no longer ambiguous.
